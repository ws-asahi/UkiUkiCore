# UkiUkiduino

**Arduino Uno R3 互換 — AVR64DU32 / USB-C**

UkiUkiduino は最新の8ビットマイコンを使用した Arduino Uno R3 互換ボードです。
USB ネイティブな `AVR64DU32` を搭載し、Uno R3 がUSB接続に追加チップを必要としたのに対し、マイコン単体でUSBによりPCやタブレットと直接つながります。

> **状態: 開発中（Rev 0.1）。** ピン定義・ブートローダは変更される可能性があります。

## Uno R3 / Leonardo との比較

UkiUkiduino の比較対象となる Arduino Uno R3 は 旧世代の8ビットマイコン **ATmega328P**（ネイティブUSB なし）を搭載しています。
また同じく旧世代の Arduino Leonardo が搭載する **ATmega32U4** はネイティブUSBを搭載していますがブートローダーで多くのプログラムメモリを消費します。
AVR64DU32 は新世代の **AVRxt コア**で、USB 内蔵・クロック・メモリ・周辺機能の多くが強化されています。


| 項目 | UkiUkiduino (AVR64DU32) | Arduino Uno R3 (ATmega328P) | Arduino Leonardo (ATmega32U4) |
|------|--------------------------|------------------------------|------------------------------|
| コア | AVRxt（命令タイミング改善） | 旧来 AVR | 旧来 AVR |
| 動作クロック | 24 MHz（内蔵オシレータ、水晶不要） | 16 MHz（外部水晶/セラロック） | 16 MHz（外部水晶/セラロック） |
| **USB** | **マイコン内蔵**（変換チップ不要でHID対応可） | なし（基板上にUSB通信チップが必要） | マイコン内蔵（CDC/HID/MIDI、変換チップ不要でHID対応可）
| Flash | **64 KB** | 32 KB | 32 KB |
| SRAM | **8 KB** | 2 KB | 2 KB |
| EEPROM | 256 B | **1024 B** | **1024 B** |
| ADC | **10-bit / 19 ch** | 10-bit / 6 ch | 10-bit / 12 ch |
| 内部基準電圧 | **1.024V / 2.048V / 2.5V / 4.096V** | 1.1V | 2.56V |
| タイマ | 8/8/8-bit | 8/16/8-bit | **8/16/16/10-bit** |
| SPI | **2 (専用ピンヘッダと D11-13 共有 + D0-2)**  | 1(専用ピンヘッダとD11-13共有) | 1(専用ピンヘッダに独立) |
| CCL（論理ブロック） | **2 LUT** | なし | なし |
| AC（アナログ比較） | **1 AC** | なし | なし |
| イベントシステム | **3 ch** | なし | なし |
| 割り込み | **21ch（無制限）** | 2ch（制限あり） | 5ch（制限あり） |


### 性能上の主な利点

- **USB がマイコン内蔵** — Uno R3 は USB-シリアル変換用の別チップを基板に載せていましたが、UkiUkiduino は Leonardo と同じく不要。`Serial` がそのまま USB CDC 仮想シリアルとなり、USB HID（キーボード/マウス）や USB-MIDI にもなれます。
- **クリスタルレス** — USB 用 48 MHz（CLK_USB）は内蔵 PLL が生成し、USB 接続時は USB の SOF に同期して自動調整されます。水晶振動子が不要で部品点数とコストを削減しています。
- **クロックと処理速度** — 24 MHz 動作（Uno の 16 MHz 比で 1.5 倍）に加え、AVRxt コアは一部命令のタイミングが改善されています。
- **記憶領域** — Flash 2 倍（64 KB）、SRAM 4 倍（8 KB）。
- **大量のアナログ入力** - D7 以外の全 GPIO でアナログ入力の読取りが可能です。
- **追加のPWM出力** - D12でPWM出力が可能です。Uno R3には無く、LeonardoではD13にあったPWMが移動しています。
- **新世代の周辺機能** — CCL（4 論理ブロック）とイベントシステム（6 チャネル）、アナログ比較器（1 チャネル）により、CPU を介さないハードウェア信号処理が可能。ATmega328P と ATmega32U4 にはいずれもありません。
- **タイマ利用** - Servo や Tone 使用時には TCB1 を使用するためD3 ピンの PWM が無効化されます（Uno R3 では D3 と D11 、Leonardo では D5 が無効）。
- **2つ目のSPI** - USART0 を使用して2つ目の SPI（SPI1）を使用できます。D0-D2を使用するためSerial1と排他利用になります。
- **割り込み** - 全GPIOでエッジ/レベルの制約無く無制限に割り込み処理を実行可能です。1クロック未満の入力も検知します。
- **ユーザーボタン搭載** — 標準的な Arduino ボードには無いオンボードのユーザーボタン（D20 / BNT_BUILTIN）を搭載。追加部品なしで入力プログラムを試せます。

### 留意点

- **EEPROM 容量は旧世代 AVR のほうが大きい**（11024 B 対 256 B）。多くの不揮発データを保存する用途では保存方法の見直し（User Row やフラッシュの活用）が必要になる場合があります。
- **D7 はアナログ入力非対応**（PA0 は ADC チャネルを持ちません）。このためアナログ別名 **A13 は欠番**です。
- **Tx / Rx LED が無い**（ピン不足による削減）。Serial に対する出力をボード上でモニターすることはできません。
- **TWI（I2C）は UnoR3 互換**。Leonardo のピン配列（D2/D3）とは一致しません。

## ピンマッピング

Arduino Uno R3 と同じ番号付け（D0–D13、A0–A5）です。
A0–A5 はデジタル D14–D19 、D0～D13 は アナログ A6-A19 を兼ねます。（ただし D7 / A13 のみアナログ入力不可）

| D# | MCU | アナログ別名 | ADC ch | 主な機能 |
|----|-----|--------------|--------|----------|
| D0 | PA5 | A6 | AIN25 | **RX**（Serial1) / MOSI（SPI1） |
| D1 | PA4 | A7 | AIN24 | **TX**（Serial1) / MISO（SPI1） |
| D2 | PA6 | A8 | AIN26 | XCK（Serial1） / SCK（SPI1） |
| D3 | PF5 | A9 | AIN21 | PWM |
| D4 | PF4 | A10 | AIN20 |  |
| D5 | PD0 | A11 | AIN0 | PWM / CCL（LUT2-IN0） |
| D6 | PD1 | A12 | AIN1 | PWM / CCL（LUT2-IN1） |
| D7 | PA0 | — | — |  |
| D8 | PA7 | A14 | AIN27 | EVOUTA / AC OUT |
| D9 | PD2 | A15 | AIN2 | PWM / CCL（LUT2-IN2）/ AC AINP0 / EVOUTD |
| D10 | PD3 | A16 | AIN3 | PWM / CCL（LUT2-OUT）/ AC AINN0  |
| D11 | PD4 | A17 | AIN4 | PWM / **MOSI**（SPI） |
| D12 | PD5 | A18 | AIN5 | PWM / **MISO**（SPI） |
| D13 | PD6 | A19 | AIN6 | **LED_BUILTIN** / **SCK**（SPI） |
| D14 | PF0 | A0 | AIN16 | CCL（LUT3-IN0） |
| D15 | PF1 | A1 | AIN17 | CCL（LUT3-IN1） |
| D16 | PF2 | A2 | AIN18 | CCL（LUT3-IN2） |
| D17 | PF3 | A3 | AIN19 | CCL（LUT3-OUT） |
| D18 | PA2 | A4 | AIN22 | アナログ A4 / **SDA**（I2C） |
| D19 | PA3 | A5 | AIN23 | アナログ A5 / **SCL**（I2C） |
| D20 | PA1 | — | — | **BTN_BUILTIN** （物理ピンなし） |
| AREF | PD7 | — | — | **VREFA**（外部基準電圧入力） |

**ヘッダの専用ピン:** ICSP（PD4/PD5/PD6 = MOSI/MISO/SCK + RESET）／Power ヘッダ 1 番ピン = **UPDI**（PF7）。
**内部・非ヘッダピン:** PC3（LED_BUILTIN ドライバ, AIN31）／PF6（RESET）。

### シリアルポート

| オブジェクト | 実体 | ピン | 備考 |
|--------------|------|------|------|
| `Serial` | USB CDC | USB-C | シリアルモニタ（仮想 COM）。日常の `Serial.print()` は Uno と同じ感覚で使えます |
| `Serial1` | USART0 | D0(RX) / D1(TX) | Uno R3 の D0/D1 ハードウェア UART |

> `Serial` は USB CDC なので、シリアルモニタ用途は Uno R3 と同じく `Serial` を使います。外部機器と D0/D1 でやり取りする場合のみ `Serial1` を使ってください。

### SPI（ホスト）

**SPI**
| 信号 | ピン |
|------|------|
| MOSI | D11 |
| MISO | D12 |
| SCK | D13 |

SPI は Uno R3 と同じく D11–D13 および ICSP ヘッダ共用で配置されています。

**SPI1**
| 信号 | ピン |
|------|------|
| MOSI | D0 |
| MISO | D1 |
| SCK | D2 |

SPI1 はボード独自の拡張により USART0 を利用して追加のSPIデバイスへ接続できます。


### I2C（Wire）

| 信号 | ピン |
|------|------|
| SDA | A4 |
| SCL | A5 |

Uno R3 と同じ A4/A5 に配置されています（TWI0 既定位置）。
通常の `Wire.begin()` でそのまま使えます。

### PWM（`analogWrite()`）

- **D3, D5, D6, D9, D10, D11, D12**
- Uno R3 に対して D12 へ PWM 機能が追加されています。

> **Uno R3 / tone() との関係:** `tone()` はタイマー **TCB1** を使います。
TCB1 は D3 PWM と共用のため、`tone()` 実行中は D3 の PWM のみ使用不可になります（他の PWM は動作継続）。
これは Uno R3 の Timer2（`tone()` 実行中は D3 および D11 が無効）に相当する挙動です。

### アナログ入力

- D7 を除く各デジタルピン（D0–D19）も ADC チャネルを持ち、A6–A19 として参照可能（**A13 は欠番**）
- **AREF 端子（PD7 / VREFA）** から外部基準電圧を入力できます
- 内部基準電圧は **1.024V / 2.048V / 2.5V / 4.096V** の4種類（Uno R3 / Leonardo は1つずつ）
- **USB保護に理想ダイオードを採用** - 多数の保護を与えると同時に MCU に対して USB バスパワー駆動時でもピュアな 5V 電圧を供給できるのでアナログ入力値が安定します。

## クロック

UkiUkiduinoは**クリスタルレス構成**です。
システムクロックは内蔵オシレータ（OSCHF）、USB 用の 48 MHz（CLK_USB）は内蔵 PLL48M が生成し USB の SOF に同期して自動調整されます。
Leonardo では USB クロック生成のために水晶発振器が必須でしたが、UkiUkiduino では無くても安定した USB 通信が可能です。

---

## 電源

UkiUkiduino は **2 系統の電源入力**を持ち、いずれからでも 5V を得られます。両方を同時に接続しても安全です。

- **USB-C（5V）:** 理想ダイオードで保護しています。逆流防止に加え、突入電流制限、過電流フォールドバック制限、サーマルシャットダウンを内蔵し、ホストポートを保護します。
- **DC ジャック（7–12V、推奨 7–9V）:** φ5.5/2.1mm の DC ジャックから入力し、ダイオードで逆接続保護後、リニアレギュレータで 5V を生成します。リニア方式のため入力電圧が高いほど発熱が増えます。**大きな負荷（数百 mA 以上）を駆動する場合は 7–9V での使用を推奨**します。
- **USB保護に理想ダイオードを採用** - 一般的な Arduino は USB 電源ラインの保護をダイオードで行うため5V電圧が低下しますが、UkiUkiduinoは理想ダイオードXC8110を採用することでUSB電源の電圧低下がほぼありません。
- **3.3V（シールドピン用）:** 基板上のリニアレギュレータで 5V から 3.3V を生成します。

## LED とスイッチ

| 部品 | 接続 | 用途 |
|------|------|------|
| 電源 LED | 電源（赤） | 通電表示 |
| **LED_BUILTIN** | **D13**（黄 / 砲弾型） | オンボード LED |
| リセット | RESET | タクトスイッチ |
| **BTN_BUILTIN** | **D20** | オンボードボタン |

`LED_BUILTIN` は **D13** です。D13 は SPI SCK と共用のため、Uno R3 と同様に SPI 使用中は LED が SCK のトラフィックで点滅します。
**BTN_BUILTIN（D20）** は 1kΩ でプルダウンされており、**押すと HIGH** になります。プルアップ設定は不要で、`digitalRead(BTN_BUILTIN)` がそのまま使えます。

## 書き込み

1. ボードを USB で接続します。
2. Arduino IDE からスケッチを書き込みます。書き込み開始時に **1200bps タッチ**が行われ、USB CDC ブートローダへ自動遷移します。
3. 自動遷移しない場合は、**リセットボタンのダブルタップ**でブートローダに入れます。

初回のみ、または USB ブートローダを書き込み直す場合は、UPDI プログラマ（PICkit 4/5、Atmel-ICE、SerialUPDI 等）を **Power ヘッダの UPDI ピン** に接続して書き込みます。
GND・5V も同じ Power ヘッダから取れるため、専用パッドや治具なしで書き込めます。
> ※ 旧世代 AVR と異なり SPI からブートローダーやスケッチの書き込みはできません。

## ソフトウェア互換性（Arduino Uno R3 / Leonardo）

UkiUkiduino は Uno R3 / Leonardo からの移植の手間を最小化することを目指しています。
`Serial` は USB CDC、I2C は A4/A5、SPI は D11–D13、`LED_BUILTIN` は D13、AREF は外部基準電圧入力と、Uno R3 の作法がそのまま通用します。

> ※ レジスタを直接読み出す動作に関しては変更が必要です。
